# 🔍 주제

거리와 길의 경사 선택, 사용자 피드백을 반영해 개인 맞춤형 러닝 경로를 추천하는 서비스

|항목|내용|
|---|---|
| 팀명칭 | 코드의 정상화 |
| 프로젝트명 | 거리와 길의 경사 선택, 사용자 피드백을 반영해 개인 맞춤형 러닝 경로를 추천하는 서비스 |
| 키워드 | 경로추천, 피드백, GraphHopper |
| 무엇을 만들고자 하는가 | 사용자가 출발지와 목적지, 거리, 길의 경사를 선택하면 길의 정보를 실시간으로 반영하여 러닝 경로를 추천하고 사용자 피드백을 통해 추천 경로를 개선함 |
| 고객 | 이현우(32세, 서울 강남구 거주)는 IT 회사에서 데이터 분석가로 근무하는 직장인으로, 주 3-4회 러닝을 즐기는 열정적인 러너이다. 평균 5-10km의 러닝을 선호하며, 시간당 5-6km의 속도로 1시간 안팎의 러닝을 주로 계획한다. 건강 관리와 체력 단련, 스트레스 해소를 목표로 운동하며, 특히 서울 내 다양한 경로를 탐색하고 새로운 장소를 달리는 즐거움을 느낀다.<br>현우는 출근 전에 러닝을 하는 경우가 많아 러닝시간뿐만 아니라 출발부터 러닝을 완료한 후 집에 돌아오기까지 걸리는 시간을 알 수 있기 원하고, 무릎 건강을 고려해 경사도가 낮은 평탄한 길을 선호한다. 기존의 러닝 앱들은 이러한 개인적인 니즈를 세부적으로 반영하지 못해 아쉬움을 느끼고 있다. 또한 반복적인 경로로 인해 러닝에 대한 흥미가 점점 떨어지고 있어, 자신만의 맞춤형 경로 추천 서비스를 간절히 바라고 있다.|
| Pain Point | 원하는 거리만큼 러닝을 하고 싶지만 길을 잘 몰라 직접 돌아다니며 거리와 경사를 확인해야 함 |
| 솔루션 | React Native<br>Django<br>Python<br>PostgreSQL<br>GraphHopper<br>Scikit-Learn |
| 오래 축적한 데이타의 잠재적 가치 | 사용자 패턴과 보편적으로 선택되는 경로 패턴의 분석을 통해 초기 런칭 때보다 시행착오를 덜 겪고 개인 맞춤형 경로를 추천해줄 수 있음 |
| 기대성과 | 사용자가 원하는 시간 또는 거리만큼 러닝할 수 있으므로 불필요한 에너지 소모를 줄일 수 있음<br>-사용자가 기존 러닝 코스보다 다양한 코스로 러닝을 할 수 있게 되므로 러닝 빈도가 늘어나는 효과를 기대할 수 있음 |
